# Contributing to Amplenote Plugin Manager
We welcome contributions!

## How to Contribute
1.  **Fork** the repository.
2.  **Clone** your fork.
3.  **Branch** off `main` for your feature: `git checkout -b feature/amazing-feature`.

## Workflow
1.  Use `esbuild.js` to manage your build.
2.  If adding a new tool, please add it to the `lib/` or create a new `anp-*` directory if intended as an example.
3.  Run `npm run build` to verify the bundler works.

## Development Standards
-   **Style**: Use modern ES6+ Javascript.
-   **Structure**: Keep the codebase modular.
-   **Documentation**: Update README if you change build logic.

## Pull Requests
-   Provide a clear description of changes.
-   Link to any relevant issues.
